01

제품 개요

미래사업팀이 새로운 건설사업관리 용역을 수주할 때 작성하는 제안서는, 현장마다 조건이 달라 매번 처음부터 구성해야 합니다. 이 시스템은 현장 정보를 입력하면 AI가 사업여건을 분석하고, 과거 제안서 아이템 DB에서 목차별로 최적 슬라이드를 자동 배치하여 PPTX 초안을 생성합니다.

핵심 가치: 제안서 초안 작성에 걸리는 시간을 수 일 → 수 시간으로 단축. 완성된 결과물이 아닌 "편집 가능한 좋은 초안"을 빠르게 확보합니다.

연관 시스템

시스템 역할 이 MVP와의 관계
proposal-db-mvp 과거 제안서 이미지 키워드 DB 아이템 소스 DB — 이 시스템이 소비
이 MVP 초안 자동생성 + PPTX 출력 신규 구축
AI (Adobe Illustrator) 최종 디자인 편집 PPTX를 받아 후속 작업
02

해결하려는 문제

현재 Pain Point 이 시스템의 해결 방법
현장마다 과거 유사 자료를 일일이 찾아 복사·붙여넣기 AI가 사업여건 분석 후 DB에서 자동 검색·배치
목차 구성을 매번 처음부터 검토 표준 목차 템플릿 + 현장 특성에 따른 가중치 조정
어떤 자료가 어디 있는지 기억에 의존 proposal-db-mvp 키워드 DB 활용, 자동 연결
초안 완성까지 장시간 소요 입력 후 수 분 이내 PPTX 초안 생성
03

사용자 정의

주 사용자: 미래사업팀 담당자

  • 건설·CM 도메인 전문가이지만 개발·IT 비전문가
  • 공고문(PDF/Word)을 받아 제안서를 작성하는 업무 담당
  • 빠른 초안 생성 후 AI(일러스트레이터)로 후속 편집
UI 원칙: 개발자 용어, 설정 화면 없음. 모든 인터랙션은 "다음 단계"가 명확한 위저드 형태로 안내합니다.

MVP 범위 외 사용자

  • 로그인·권한 관리 없음 (단일 팀 내부 사용)
  • DB 관리자(아이템 업로드)는 proposal-db-mvp 시스템 사용

① 새 제안서 "새 제안서 시작" 클릭 → 현장명 입력 제안서 레코드 생성 ~1분
② 현장 정보 구조화 폼 입력 + RFP 파일 업로드 (선택) 입력값 저장, 파일 파싱(OpenAI) 후 핵심 정보 추출 5~15분
③ AI 분석 "분석 시작" 클릭 → 로딩 대기 현장 정보를 GPT-4o로 분석, 섹션별 키워드 및 가중치 생성 ~1분 (자동)
④ 초안 생성 분석 결과 확인 (섹션별 추천 키워드) proposal-db-mvp에서 섹션별 아이템 검색·배치 ~30초 (자동)
⑤ 웹 검토 슬라이드 순서 변경, 아이템 교체, 삭제, 추가 변경사항 즉시 저장 자유 (10분~수 시간)
⑥ PPTX 내보내기 "PPTX 다운로드" 클릭 서버에서 PPTX 생성 → 파일 다운로드 ~30초
05

기능 요구사항

F1. 현장 정보 입력

항목우선순위설명
구조화 폼 P1 현장명, 발주처, 위치, 공사 규모(금액·면적·기간), 공종, 특이사항, 요구 핵심역량 등 입력
RFP 파일 업로드 P1 PDF 또는 Word 파일 업로드 → GPT-4o로 핵심 정보 자동 추출 → 폼에 자동 채우기 제안
추가 자료 업로드 P2 현장 사진, 위치도 등 참고 이미지 첨부 (슬라이드에 직접 삽입 가능)
임시 저장 P1 입력 중 자동 저장, 나중에 이어서 작업 가능

F2. AI 사업여건 분석

항목우선순위설명
현장 특성 분류 P1 입력 정보 기반으로 공종(토목/건축/복합), 규모(대형/중형/소형), 특수 요소(환경·안전·BIM 강조 여부 등) 판단
섹션별 검색 키워드 생성 P1 각 목차 섹션에 대해 이 현장에 맞는 검색 키워드 3~5개 자동 생성
분석 결과 요약 표시 P2 AI가 파악한 현장 특성 요약을 사용자에게 보여주고 수정 허용

F3. 목차 및 아이템 배치

항목우선순위설명
표준 목차 템플릿 P1 건설CM 제안서 표준 목차(기존 17개 섹션 기준)를 기본 제공. 사용자가 섹션 추가·삭제·순서 변경 가능
자동 아이템 배치 P1 proposal-db-mvp DB에서 섹션별로 가장 적합한 아이템 1개 자동 배치. 추천 후보 3개 함께 제시
아이템 수동 교체 P1 배치된 아이템을 클릭하면 동일 섹션의 후보 아이템 패널 표시 → 선택하여 교체
아이템 추가/삭제 P1 섹션 내 아이템 추가(DB 검색 모달) 또는 삭제
슬라이드 순서 변경 P2 드래그 앤 드롭으로 슬라이드 순서 조정
DB 직접 검색 P2 키워드로 proposal-db-mvp를 직접 검색해 원하는 아이템 추가

F4. 웹 미리보기

항목우선순위설명
섬네일 그리드 P1 전체 슬라이드를 썸네일 격자로 표시. 섹션 구분선 포함
슬라이드 상세 보기 P1 클릭 시 원본 이미지 확대 표시
섹션 접기/펼치기 P2 목차 섹션 단위로 접어서 전체 구조 파악 용이

F5. PPTX 내보내기

항목우선순위설명
PPTX 생성 및 다운로드 P1 현재 배치된 아이템 이미지를 슬라이드별로 삽입하여 PPTX 생성
섹션 구분 슬라이드 P2 목차 섹션 시작 시 구분용 타이틀 슬라이드 자동 삽입
표지 슬라이드 P2 현장명·발주처 등 기본 정보가 포함된 표지 슬라이드 자동 생성

F6. 제안서 관리

항목우선순위설명
제안서 목록 P1 작성 중인 제안서 목록 (현장명, 생성일, 진행 상태)
이어서 작업 P1 중단 후 재접속해도 진행 상태 유지
제안서 복사 P3 유사 현장 제안서를 복사해 기반으로 사용

PPTX 출력 규격

AI(Adobe Illustrator) 편집 전제: 생성된 PPTX는 중간 결과물입니다. PPTX → PDF 내보내기 → Illustrator에서 열어 최종 디자인 편집하는 워크플로우를 지원합니다.
항목규격이유
슬라이드 사이즈 297mm × 210mm (A4 가로) 인쇄·PDF 호환, AI에서 정확한 치수 편집
아이템 이미지 배치 슬라이드 전체 채움 (Full-bleed) 원본 이미지를 최대한 보존
텍스트 요소 아웃라인 변환 없이 텍스트 박스로 삽입 AI에서 텍스트 편집 가능
섹션 구분 슬라이드 섹션명 + 배경색 텍스트 슬라이드 목차 구조 파악, AI 편집 시 구분점
이미지 해상도 원본 PNG 그대로 삽입 (최소 1000px) 화질 손실 없이 AI 편집
메타데이터 슬라이드 노트에 아이템 ID, 섹션명 기록 AI 편집 후 추적 가능

MVP 범위 (In / Out)

✅ MVP에 포함

  • 현장 정보 텍스트 입력 폼
  • RFP PDF/Word 업로드 + 자동 파싱
  • GPT-4o 기반 사업여건 분석
  • proposal-db-mvp 연동 아이템 검색
  • 목차별 아이템 자동 배치
  • 웹 미리보기 (썸네일 그리드)
  • 아이템 교체 / 추가 / 삭제
  • PPTX 내보내기 (A4 가로)
  • 제안서 목록 / 이어서 작업

❌ MVP 제외 (추후)

  • 로그인 / 사용자 권한 관리
  • 텍스트 직접 편집 (슬라이드 내)
  • 실시간 공동 편집
  • 아이템 DB 관리 (→ proposal-db-mvp에서)
  • PDF 직접 출력
  • 버전 이력 / 롤백
  • 제안서 공유 링크
  • OCR 기반 이미지 내 텍스트 추출
  • 모바일 최적화

미결 사항

Q1. 표준 목차 구조 확정

  • 건설CM 제안서의 표준 섹션 수·명칭을 팀 실무 기준으로 확정 필요
  • 확정된 목차는 proposal-db-mvp의 taxonomy와 동일하게 맞춰야 아이템 검색 정확도 향상

Q2. proposal-db-mvp 연동 방식

  • 별도 Supabase 프로젝트이므로 Service Role Key 공유 또는 API 래퍼 구성 필요
  • 아이템 이미지 URL: 원본 참조 vs. 이 시스템 Storage에 복사

Q3. 섹션별 아이템 수

  • 한 섹션에 아이템을 몇 장까지 배치할 수 있는지 (기본값, 최대값)
  • 섹션별로 차이가 있는지 (예: "조직 구성"은 1장, "공정관리"는 다수)

Q4. RFP 파일 형식

  • 주로 사용되는 공고문 형식 (PDF, HWP, Word)
  • HWP 지원 여부 — 별도 변환 도구 필요

Q5. PPTX → AI 워크플로우

  • PPTX를 PDF로 내보내어 AI에서 여는 방식인지, 직접 PPTX를 AI로 여는 방식인지
  • AI에서 편집 후 다시 시스템으로 가져오는 흐름이 필요한지
